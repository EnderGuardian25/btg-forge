BTG Forge · System Reference

Gate-driven, spec-driven
developer for Claude Code.

Every unit of work travels one rail — from a plain-English request to a reviewed pull request — passing six honest gates. Lightweight by default; it escalates to heavier BMAD planning only when complexity earns it, and enforces test-first discipline in the middle.

init → specify [G0] → clarify → plan [G1/G2] → tasks → implement [G3] → verify [G4] → archive → pr-review [G5]
01

The pipeline

All ten commands on a single spine. Gates hang to the right as checkpoints; the BMAD escalation branches left off plan when triage returns HIGH. The read-only brainstormer feeds ideas into specify and plan up front. Connectors are strictly horizontal & vertical.

BTG Forge pipeline diagram A vertical spine of commands — init, specify, plan, tasks, implement, verify, archive, pr-review — with gates G0 through G5 as checkpoints on the right, a conditional clarify loop on gate G0, an advisory analyze step feeding gate G4, a BMAD escalation branch (analyst, pm, architect, po) on the left off the plan command, and a read-only brainstormer agent feeding ideas into specify and plan.
command flow BMAD tier (HIGH) brainstormer (ideation) conditional / advisory  gate = PASS · CONCERNS · FAIL
A FAIL stops the pipeline. It may only proceed if a human writes a Justification: line into changes/<feature>/gates.md — the complexity-tracking escape hatch.
02

The architecture

Five layers packaged under .claude/ and .forge/. Commands orchestrate; agents do the specialized work; skills encode discipline; hooks bracket the session; the .forge/ state folder is the single source of truth every stage reads and writes.

BTG Forge architecture diagram Five stacked layers: hooks (session-start, pre-commit); ten forge commands; agents grouped into the shared gate evaluator, the brainstormer ideation agent, the BMAD quartet, the TDD pair, the review trio, and the standalone mcp-connector utility; four skills; and the .forge state folder holding the constitution, preferences, learnings, patterns, living specs, per-change artifacts and the archive.
